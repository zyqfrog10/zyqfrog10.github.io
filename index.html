How to update R without re-installing packages on Mac OS

R is a fast developing language/software. I am glad to update it as often as possible to enjoy new features. However, updating R is not always an enjoyable thing when you have a bunch of packages that do not migrate to your new version automatically. Here are some tips that may save us some time compared to manually downloading everything again. (Note: Since I am using Mac, the tips are Mac based.)

Install new version of R

Just follow the steps as you first time install R:

  1. Go to http://www.r-project.org

  2. Click on 'CRAN', 'CRAN mirror', or 'download R'

  3. Choose a CRAN

  4. Click on 'Download R for XXX', where XXX is your operating system

  5. Download the new version of R

  6. Follow the installation procedure for your operating system

Copy your packages to the new version R

Now you have a new R, but only have base packages in the library. Let's enrich the new library without downloading packages one by one.

  1. Copy the packages from the old R library into the new one on Mac OS Example:

        cp -r -n /Library/Frameworks/R.framework/Versions/3.1/Resources/library/  /Library/Frameworks/R.framework/Versions/3.2/Resources/library/

    Note: -rn is necessary to copy entire folder without overwriting existing packages in the new destination.

  2. Update your packages

        update.packages(checkBuilt=TRUE)

    At this step, I encountered a warning message after downloading failed: "Warning: cannot remove prior installation of package 'XXX'", where XXX is any package on the top of the list. Don't know why this happened. The good thing is re-installing new version R will fix it (copied packages are still there, you don't have to repeat the copy step.).

If you use Rstudio

Simply open up Rstudio, new version R is automatically loaded. In my case, updating to R3.2, I got an error message like 'Error in tools:::httpdPort <= 0L : comparison (4) is possible only for atomic and list types.' Then I updated my Rstudio to the latest version and the problem got solved by itself.

Alternative solution

You may go the link http://zvfak.blogspot.com/2012/06/updating-r-but-keeping-your-installed.html for an alternative solution. The following is a summary.

  1. Build your own library path that should be indicated in ~/.Renviron file

        mkdir ~/Rlib

  2. Create a .Renviron file in your home directory with the following line in the content

        R_LIBS=~/Rlibs

  3. Update R as usual and packages in ~/Rlibs are intact

  4. Update packages as you normally do with update.packages()

Enjoy your new R!
